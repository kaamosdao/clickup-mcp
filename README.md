# clickup-mcp

MCP-сервер для получения и анализа time entries из ClickUp. Отдаёт сырые записи, сгруппированный отчёт по времени и описание структуры полей.

## Установка

```bash
git clone git@github.com:kaamosdao/clickup-mcp.git
cd clickup-mcp
npm install
```

Требуется Node.js с поддержкой ESM и `fetch` (Node 18+).

## Конфигурация

Сервер читает два обязательных параметра из переменных окружения:

| Переменная     | Описание                                  |
| -------------- | ----------------------------------------- |
| `CLICKUP_API`  | Personal API token ClickUp (`pk_...`)     |
| `TEAM_ID`      | ID команды (workspace) в ClickUp          |

Токен берётся в ClickUp: **Settings → Apps → API Token**.

### Подключение к Claude Code

Добавьте сервер в конфиг MCP (`~/.claude.json` или через `claude mcp add`):

```json
{
  "mcpServers": {
    "clickup": {
      "type": "stdio",
      "command": "node",
      "args": ["/абсолютный/путь/clickup-mcp/clickup-mcp.mjs"],
      "env": {
        "CLICKUP_API": "pk_xxxxxxxx",
        "TEAM_ID": "0000000"
      }
    }
  }
}
```

## Инструменты

### `get_time_report`

Сгруппированный отчёт по записям времени. Для каждой группы — итоговое время и «полные» (×1.25), затем детали по каждой записи: интервал начало→конец, длительность, задача, борда в `[скобках]`, billable/unbillable.

**Чистые и грязные часы:**

- **чистые** — реальные часы, залогированные в ClickUp (фактическая длительность);
- **грязные** — чистые, умноженные на 1.25 (в отчёте обозначены как «полные (×1.25)»).

| Параметр     | Описание                                                                                   |
| ------------ | ------------------------------------------------------------------------------------------ |
| `date`       | Дата `YYYY-MM-DD` (по умолчанию — сегодня)                                                  |
| `start_date` | Начало диапазона `YYYY-MM-DD`                                                               |
| `end_date`   | Конец диапазона `YYYY-MM-DD`                                                                |
| `group_by`   | `folder` (по умолчанию), `list`, `user`, `billable`, `task`, `space`                       |

Отчёт нужно показывать пользователю целиком и дословно, без сокращений и опускания полей.

### `get_time_entries`

Сырые time entries за период — полный JSON-массив для кастомного анализа и фильтрации.

| Параметр     | Описание                                  |
| ------------ | ----------------------------------------- |
| `date`       | Дата `YYYY-MM-DD` (по умолчанию — сегодня) |
| `start_date` | Начало диапазона `YYYY-MM-DD`             |
| `end_date`   | Конец диапазона `YYYY-MM-DD`              |

### `get_schema`

Описание полей time entry и маппинг терминов (проект = folder, борда = list, задача = task, billable). Полезно вызвать перед анализом, чтобы понять структуру данных.

## Маппинг терминов

| Термин              | Поле                            |
| ------------------- | ------------------------------- |
| проект (project)    | `task_location.folder_name`     |
| борда (board)       | `task_location.list_name`       |
| задача (task)       | `task.name`                     |
| платная/бесплатная  | `billable` (`true`/`false`)     |
